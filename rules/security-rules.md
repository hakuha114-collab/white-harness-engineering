# 安全合规红线

> L0 级不可逾越红线，违反即终止流程

## 1. 数据安全

### 1.1 敏感数据处理

- [ ] **绝对禁止** 将密钥、Token、密码硬编码在代码中
- [ ] **绝对禁止** 在日志中输出敏感信息（密码、Token、个人隐私数据）
- [ ] **绝对禁止** 在前端代码中暴露后端内部接口地址
- [ ] **绝对禁止** 将敏感数据存储在未加密的 Cookie/LocalStorage 中
- [ ] 所有敏感配置必须通过环境变量或密钥管理系统注入

### 1.2 数据传输

- [ ] 所有外部通信必须使用 HTTPS
- [ ] API 接口必须进行身份认证和授权校验
- [ ] 敏感数据传输必须加密
- [ ] 禁止在 URL 参数中传递敏感信息

### 1.3 数据存储

- [ ] 用户密码必须使用 bcrypt/argon2 等算法加密存储
- [ ] 个人隐私数据必须脱敏后才能存储和展示
- [ ] 数据库访问必须使用参数化查询，防止 SQL 注入
- [ ] 敏感数据必须有访问审计日志

## 2. 代码安全

### 2.1 输入校验

- [ ] 所有用户输入必须进行类型校验和长度限制
- [ ] 所有用户输入必须进行 XSS 过滤
- [ ] 所有文件上传必须校验类型、大小和内容
- [ ] 所有 URL 跳转必须校验白名单，防止 SSRF

### 2.2 代码执行

- [ ] **绝对禁止** 使用 `eval()`、`exec()` 等动态代码执行
- [ ] **绝对禁止** 直接拼接 SQL 语句
- [ ] **绝对禁止** 在服务端渲染用户输入的 HTML
- [ ] 命令行参数必须转义，防止命令注入

### 2.3 依赖安全

- [ ] 所有依赖必须通过安全审计
- [ ] 禁止使用有已知 CVE 漏洞的依赖
- [ ] 依赖更新必须进行兼容性和安全性测试
- [ ] 锁定依赖版本，防止供应链攻击

## 3. 访问控制

### 3.1 认证与授权

- [ ] 所有 API 必须有认证机制
- [ ] 必须实现最小权限原则
- [ ] 必须实现 RBAC 或 ABAC 权限模型
- [ ] Token 必须设置合理的过期时间
- [ ] 关键操作必须有二次确认

### 3.2 会话管理

- [ ] Session 必须设置合理过期时间
- [ ] 登录失败必须有频率限制
- [ ] 必须支持强制登出功能
- [ ] 并发会话必须有控制策略

## 4. 基础设施安全

### 4.1 容器安全

- [ ] 容器镜像必须使用官方基础镜像
- [ ] 容器必须以非 root 用户运行
- [ ] 容器端口必须最小化暴露
- [ ] 容器必须设置资源限制

### 4.2 网络安全

- [ ] 服务间通信必须使用内网
- [ ] 必须配置网络隔离策略
- [ ] 必须配置防火墙规则
- [ ] 必须配置 DDoS 防护

## 5. AI 特有安全

### 5.1 Prompt 安全

- [ ] **绝对禁止** 将用户输入直接拼入系统 Prompt
- [ ] 必须对 Prompt 输入进行边界检查
- [ ] AI 产出必须经过安全过滤后方可使用
- [ ] 禁止 AI 访问未授权的系统资源

### 5.2 Agent 安全

- [ ] Agent 执行外部命令必须经过审批
- [ ] Agent 文件系统访问必须在沙箱内
- [ ] Agent 网络请求必须经过代理和审计
- [ ] Agent 必须有操作频率限制

---

**校验脚本**: `scripts/check-security/`
**违反处理**: L0 级别，立即终止流程，必须修复后才能重新启动
