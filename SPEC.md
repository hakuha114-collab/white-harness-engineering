# 设计规格文档 (SPEC)

> 本文档为 Harness Engineering AI 工程底座的设计规格，统一全员 AI 基准标准。

## 1. 项目概述

### 1.1 目标

搭建一套全覆盖约束规则、标准化作业流程、可审计闭环链路的 AI 工程管控底座，使公司所有软件开发环节（代码开发、修改、Review、测试、项目管理、开发规范等）在 AI 协助时，均在既定框架内高效执行，全程可控、可追溯、可迭代、可规模化。

### 1.2 适用范围

- 所有使用 AI 辅助的软件开发活动
- 所有接入公司工程体系的 AI Agent
- 所有涉及代码产出、文档产出、流程决策的 AI 行为

### 1.3 核心约束

- AI 执行任何操作前，必须先校验 Rule 合规性
- AI 产出必须经过 Scripts 门禁验收后方可推进
- 所有 AI 行为必须有审计日志，可追溯
- 任何回退/异常必须记录原因和处理结果

## 3. 非功能规格

### 3.1 性能要求

| 指标 | 要求 |
|------|------|
| 规则加载延迟 | < 500ms |
| 门禁验收耗时 | < 30s（单节点） |
| Agent 交接延迟 | < 1s |
| 全流程端到端 | 视项目复杂度，关键路径不阻塞 |

### 3.2 可靠性要求

| 指标 | 要求 |
|------|------|
| 门禁校验准确率 | ≥ 99% |
| 流程回退成功率 | ≥ 99.9% |
| 审计日志完整性 | 100% |
| MCP 集成可用性 | ≥ 99.5% |

### 3.3 安全要求

- 所有 Rule L0 级别约束不可绕过
- Agent 无权修改自身 Rule 配置
- 敏感操作必须有人工确认
- 所有 MCP 外接需鉴权

### 3.4 可扩展性

- 支持自定义 Agent 注册
- 支持自定义 Skill 开发
- 支持自定义 Scripts 编写
- 支持自定义 MCP 集成
- 支持自定义 Workflow 编排

## 4. 接口规格

### 4.1 Agent 间通信接口

```yaml
# 消息格式
Message:
  id: string          # 消息唯一标识
  from: string        # 发送方 Agent ID
  to: string          # 接收方 Agent ID
  type: enum          # handover | rollback | query | notify
  priority: enum      # urgent | high | normal | low
  payload:
    task_id: string   # 任务标识
    artifacts: []     # 产出物路径列表
    context: map      # 上下文信息
    reason: string    # 回退原因（rollback 时必填）
  timestamp: string   # ISO 8601
```

### 4.2 门禁验收接口

```yaml
# 验收请求
CheckRequest:
  gate: string        # 门禁节点名称
  task_id: string     # 任务标识
  artifacts: []       # 待校验产出物
  rules: []           # 适用的规则列表

# 验收结果
CheckResult:
  gate: string
  status: enum        # PASS | WARN | FAIL | BLOCK
  checks:
    - name: string
      status: enum    # PASS | WARN | FAIL | BLOCK
      message: string
      details: map
```

## 5. 术语表

| 术语 | 定义 |
|------|------|
| Harness | 缰绳，指对 AI 行为的约束管控体系 |
| Rule | 刚性工程底线制度，不可逾越的红线规则 |
| Skill | 标准化高频操作手册，固化可复用流程 |
| Sub Agent | 专业化分工的职能智能体 |
| Workflow | 全链路分级指挥链条，定义 Agent 间协作流程 |
| Scripts | 自动化硬性验收门禁脚本 |
| MCP | Model Context Protocol，全域工程系统外接插座 |
| Assets | 资产化沉淀，替代临时会话记忆的工程化知识 |
| dev-map | 全域工程图谱，可视化系统架构和依赖关系 |
| 门禁 | 工作流关键节点的自动验收检查点 |
| 回退 | 当前环节不通过时，返回上一环节修复 |
