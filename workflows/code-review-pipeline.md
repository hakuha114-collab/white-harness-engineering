# 代码审查流水线

> 独立代码审查流程，适用于 PR 审查、代码走查等场景

## 流程定义

```yaml
workflow:
  name: code-review-pipeline
  version: 1.0.0
  description: 独立代码审查流程
  agents:
    - pm-dispatcher
    - code-reviewer
```

## 流程图

```
[PR 提交]
    │
    ▼
[PM调度Agent] ─── 任务路由 ───▶ [代码审查Agent]
                                     │
                                     ├─ 自动化检查 (Lint + Security)
                                     │
                                     ├─ 人工级审查 (逻辑/质量/安全/架构)
                                     │
                                     │ 产出: 审查报告
                                     │ 门禁: check-review-pass
                                     ▼
                              [通过 / 不通过]
```

## 审查级别

| PR 类型 | 审查级别 | 要求 |
|---------|---------|------|
| 文档/样式变更 | L1 轻量 | 自动化检查通过 |
| 功能开发/Bug修复 | L2 标准 | 自动化 + 人工审查 |
| 核心模块/架构变更 | L3 严格 | 自动化 + 人工审查 + 技术负责人确认 |

## 时效要求

| 级别 | 首次响应 | 完成审查 |
|------|---------|---------|
| L1 | 1 小时 | 4 小时 |
| L2 | 2 小时 | 24 小时 |
| L3 | 4 小时 | 48 小时 |
